$ docker  exec  -it  09ca6feb6efc bash/env
where, 09ca6feb6efc ---> it is Id of the container

TASK 3 ->
You are a DevOps engineer in a startup company. The development team asks you to quickly deploy a simple web page that displays the message:
- Use the official httpd Docker image.
- Run the container so it is accessible on port 8080 of the host machine.
- Create or update an HTML file inside the container to display the required message.
- Verify the output using a command-line tool.
- Properly stop and remove the container after testing.

docker pull httpd
docker run -d --name my-apache -p 8080:80 httpd
docker exec -it my-apache bash -c "echo '<h1>Hello from Docker Apache!</h1>' > /usr/local/apache2/htdocs/index.html"
curl http://localhost:8080
docker stop my-apache
docker rm my-apache

TASK 4 ->
3. How would you use the docker run command with -it, -e, -v, and --name to:
- Set an environment variable APP_ENV=production.	
- Bind a local directory /app/data to /data inside the container.	
- Name the container my_app.	

docker pull ubuntu
mkdir C:\app\data
docker run -it --name my_app -e APP_ENV=production -v C:\app\data:/data ubuntu /bin/bash
echo $APP_ENV
echo "Hello this is my first program" > /data/test.txt
cat /data/test.txt
echo "Second line added" >> /data/test.txt
cat /data/test.txt
exit

Docker container commands --->
Container Lifecycle Management
- Run a container (creates and starts it) 
docker run -d -p 80:80 --name my_container nginx

- Start a stopped container 
docker start <container_id/name>

- Stop a running container 
docker stop <container_id/name>

- Restart a container 
docker restart <container_id/name>

- Pause a container 
docker pause <container_id/name>
